## Mobile-first в Next.js + TypeScript

Оптимально не вести две версии (desktop/mobile), а делать **одну адаптивную** с mobile-first.

### 1. Стили: сначала мобила, потом `md:` / `lg:`

```css
/* Mobile-first: базовые стили = мобильная версия */
.logo img { height: 32px; }
.header { padding: 0 20px; }

/* Планшеты и выше */
@media (min-width: 768px) {
  .logo img { height: 40px; }
  .header { padding: 0 40px; }
}

/* Десктоп */
@media (min-width: 1024px) {
  .logo img { height: 48px; }
}
```

С Tailwind то же самое: пишите классы без префикса для мобилы, потом `md:...`, `lg:...`.

### 2. Компоненты

- **Один компонент** (например `Header`, `Preloader`, `Footer`) с разной вёрсткой через CSS/media или условным рендером по `useMedia` только там, где реально разная структура (например, бургер vs горизонтальное меню).
- Логотип — один `<Image>` с разными `width`/`height` или `className` по брейкпоинтам.

### 3. Layout

- В `layout.tsx` задавайте базовые стили и viewport под мобилу; контейнеры без фиксированной ширины (не `width: 1440px` / `375px`), а `max-width` + `margin: 0 auto` и отступы через padding.
- Прелоадер и хедер делайте сразу адаптивными (одна разметка, стили через media/Tailwind).

### 4. Структура при переносе

- Вынести общие стили (переменные, кнопки, типографика) в глобальные стили или Tailwind-конфиг.
- Один набор секций (Hero, Services, CTA и т.д.), стилизованный с min-width медиа-запросами или утилитами `md:`/`lg:`.
- Мобильное меню (бургер + выезжающая панель) и десктопная навигация — один компонент `Navigation` с переключением по медиа-запросу или через состояние + `useEffect`/`useMediaQuery`.

Так из десктопной версии не делают отдельную «мобильную» — из одной вёрстки получается и мобильная, и десктопная за счёт mobile-first на уровне компонентов и layout.
